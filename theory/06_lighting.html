💡 Модуль 6: Освітлення та рендер

Цей модуль складається з 2 уроків і присвячений створенню професійного освітлення та фінальному рендеру сцени. Ви навчитеся налаштовувати світло, обирати рендер-двигун та отримувати якісне зображення.

💡 Урок 1. Освітлення в Cycles vs Eevee

Типи джерел світла в Blender

💡

Point

Точкове джерело – світло випромінюється в усіх напрямках з однієї точки.

Застосування: лампочка, свічка, факел
☀️

Sun

Сонячне світло – паралельні промені з нескінченно віддаленого джерела.

Застосування: денне освітлення, відкриті сцени
🔦

Spot

Прожектор – конус світла з фокусом та краєм.

Застосування: сцена, театр, ліхтарик
📐

Area

Площадкове джерело – м'яке світло з прямокутної або круглої площини.

Застосування: студійне світло, вікна

Схема трьох точок освітлення (Three-Point Lighting)

Класична схема освітлення, яка використовується в фотографії, кіно та 3D-графіці для створення об'ємного та професійного вигляду об'єкта.

HDRI – освітлення на основі зображення

HDRI (High Dynamic Range Imaging) – це зображення з високим динамічним діапазоном, яке використовується для освітлення сцени. HDRI забезпечує реалістичне навколишнє освітлення, відбиття та затінення одним кліком.

💌 Як використовувати HDRI:
  1. World Properties → Surface → Surface → Environment Texture
  2. Відкрийте HDRI файл (з polyhaven.com або інших ресурсів)
  3. Налаштуйте Strength (інтенсивність)
  4. Для контролю напрямку обертайте HDRI: Z в режимі об'єкта

💡 Порада: Використовуйте Eevee для налаштування сцени та швидких прев'ю, а Cycles для фінального рендеру. Зменшіть кількість семплів (Samples) в Cycles для тестових рендерів (64-128), потім збільште для фінального (512-2048).

🎬 Урок 2. Рендер та композитинг

Налаштування рендеру

🔧 Основні налаштування рендеру:
  • Output Properties → Resolution – роздільна здатність (1920×1080 для HD, 3840×2160 для 4K)
  • Output Properties → Frame Rate – частота кадрів для анімації (24 або 30 fps)
  • Render Properties → Sampling – кількість семплів (Render = 512-2048 для Cycles)
  • Render Properties → Light Paths – кількість відбиттів (Max Bounces = 8-12)
  • Render Properties → Performance – увімкніть GPU (Compute Device в налаштуваннях)
  • Output Properties → Output – формат файлу (PNG для зображень, FFmpeg для відео)

Композитинг в Blender

Композитинг (Compositing) – це пост-обробка рендеру для покращення зображення: додавання ефектів, корекція кольорів, глибина різкості тощо.

Формати збереження

  • PNG – основний формат для зображень (з підтримкою альфа-каналу)
  • OpenEXR – професійний формат з високою глибиною кольору (для пост-обробки)
  • FFmpeg Video – відео (MP4, AVI, MOV). Оберіть H.264 кодек для хорошого балансу якість/розмір
  • Animation (PNG sequence) – послідовність зображень для анімації (безпечніше, ніж одразу відео)
⚠️ Важливо: Для анімації завжди рендерте в PNG sequence, а потім збирайте в відео. Якщо рендер перерветься, ви не втратите вже зроблені кадри!

✅ Чекліст перед фінальним рендером

☐ Перевірте, що всі нормалі спрямовані назовні (Shift + N в Edit Mode)
☐ Переконайтесь, що всі модифікатори застосовані або залишені доцільно
☐ Перевірте UV-розгортку на спотворення (синій/червоний колір)
☐ Налаштуйте камеру – композиція, фокусна відстань, глибина різкості
☐ Увімкніть GPU Compute в налаштуваннях Cycles
☐ Встановіть оптимальну кількість семплів (512-2048)
☐ Увімкніть Denoise для чистого зображення
☐ Перевірте шлях збереження файлів
☐ Зробіть тестовий рендер малої роздільної здатності
☐ Для анімації: рендерте в PNG sequence

⚠️ Часті помилки при освітленні та рендері

❌ Помилка 1: "Чорний" рендер або дуже темне зображення
✅ Рішення: Перевірте, чи не прихована камера (вимкнена іконка камери в Outliner). Додайте більше світла або збільште Strength HDRI.
❌ Помилка 2: Шум на рендері навіть при 1000+ семплах
✅ Рішення: Увімкніть Denoise (Render Properties → Sampling → Denoise). Для Cycles використовуйте OptiX Denoise (найкраща якість).
❌ Помилка 3: Артефакти тіней в Eevee
✅ Рішення: Збільште Cube Size та Cascade Size в Shadow налаштуваннях. Додайте більше Light Probes (Irradiance Volume, Reflection Cubemap).
❌ Помилка 4: Рендер займає дуже багато часу
✅ Рішення: Зменшіть кількість семплів до 256-512. Використовуйте Light Paths → Max Bounces = 4-6. Увімкніть GPU Compute.

🔑 Ключові поняття модуля

  • Key Light / Fill Light / Back Light – три точки класичного освітлення
  • HDRI – освітлення на основі зображення високого динамічного діапазону
  • Cycles – рендер-двигун Ray tracing (фізично точний)
  • Eevee – рендер-двигун реального часу (rasterization)
  • Sampling (Samples) – кількість променів на піксель (чим більше, тим чистіше)
  • Denoise – шумодав для очищення рендеру
  • Композитинг – пост-обробка зображення
  • Depth of Field (DOF) – глибина різкості
  • Light Probe – об'єкт для збору даних про освітлення в Eevee

📦 Де взяти безкоштовні HDRI?

  • Poly Haven – найкращий безкоштовний ресурс, CC0 ліцензія
  • HDRI Haven – (тепер частина Poly Haven)
  • AmbientCG – HDRIs та PBR текстури
  • HDRI Skies – небо та хмари
